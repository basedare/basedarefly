# Posh — market validation (not a model to copy)

Posh.vip (events/nightlife; ~$95M ticket sales, $37M Series B) proves event people already run the loop BaseDare is built on: **personal link → attributable outcome → payout → repeat promoter behavior.** Their "Kickback" = public/private affiliate offers, event-level tracking links, organizer-funded commissions, instant payout. Growth stack: tracking links, QR, SMS blasts, Explore feed, playlists, Mailchimp.

**Moat distinction (don't blur it):** Posh proves **event commerce** — "who bought / attended my event." BaseDare proves **mission completion + attributed physical action** — "who performed, posted, drove people here, and what proof did the payer accept." Harder claim, more defensible, ours.

**Steal now (language/shape only — mostly things we already have):**
1. Language: "every creator gets a tracked mission link" (we have `?src=`).
2. Two-tier creator access *later*: private founding creators first → public opt-in.
3. Receipt dashboard: source link · attributed arrivals · check-ins · proof · venue co-sign.
4. Growth narrative: "the receipt sells the next venue."

**Do NOT build now:** marketplace feed, SMS blasts, affiliate automation, ticketing, full Kickback clone — all post-traction.

**Move is unchanged:** Hideaway + one creator + one receipt. Posh is conviction, not a build queue.
