# Social Connect Cross-Post Spec

## Purpose

Cross-posting is the first outward-facing growth rail in the Social Connect arc.
It should make approved BaseDare activity feel:

- credible
- premium
- place-aware
- on-chain without sounding like crypto spam

## Platform Order

1. X
2. TikTok
3. Instagram

This spec covers the X payload first.

## Trust Rules

- Strong proof language is reserved for approved BaseDare outcomes.
- Imported social history must never use the same proof framing as verified BaseDare completions.
- Open or simulation-only flows can be shared, but should not imply on-chain settlement or verified completion.

## Core Payload Elements

Every X payload should try to include:

- place name when available
- challenge title
- bounty amount when available
- target handle when relevant
- approved or live state
- BaseDare brand mention
- deep link back to the exact dare or place

## Payload Types

### 1. Live Challenge Share

Use when a dare is live and people should pile in.

Include:

- bounty amount
- challenge title
- target handle if targeted
- place name if nearby IRL
- deep link

Tone:

- high-energy
- invitational
- not over-explained

### 2. Invite / Claim Share

Use when a targeted dare is waiting for the creator to claim it.

Include:

- target handle
- challenge title if available
- bounty amount
- direct claim link

Tone:

- direct
- urgent
- creator-facing

### 3. Verified Completion Share

Use when a dare is approved or a winner is sharing the result.

Include:

- challenge title
- amount won or stake size
- place name if available
- verified / approved framing
- deep link

Tone:

- earned
- triumphant
- sharp, not cringe

## Copy Rules

- Prefer one strong hook line, one context line, one CTA.
- Avoid more than two hashtags.
- Avoid walls of text.
- Avoid fake military or dashboard language.
- Keep BaseDare tone closer to nightlife / arena / street energy.

## BaseDare Brain Note

BaseDare Brain should treat cross-posts as:

- distribution assets
- creator acquisition signals
- venue heat multipliers

It should not auto-post without explicit review or user action.
